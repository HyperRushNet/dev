hrn-game.min.js

High-performance Web Component for secure and isolated HTML5 game deployment.

Quick Start

Incorporate the library via CDN to begin using the <hrn-game> custom element.

<!-- Add to your <head> -->
<script src="https://cdn.jsdelivr.net/gh/HyperRushNet/dev@latest/lib/hrn-game_1.1_min.js"></script>

<!-- Use anywhere in your body -->
<hrn-game nr="17"></hrn-game>

Attributes

Configure the game instance behavior directly through HTML attributes.

AttributeTypeDefaultDescription
nrREQUIREDNumber-Unique Game ID from the registry.
creditsBooleantrueHyperRush watermark visibility.
redirectblockBooleantrueMutes logs & prevents frame-breaking.
extrapermissionsString""Additional sandbox flags.

Full Example

A complete standalone implementation for Game #17:

<!DOCTYPE html>
<html>
<head>
  <style>
    body { margin: 0; background: #000; height: 100vh; }
    hrn-game { width: 100%; height: 100%; display: block; }
  </style>
  <script src="https://cdn.jsdelivr.net/gh/HyperRushNet/dev@latest/lib/hrn-game_1.1_min.js"></script>
</head>
<body>
  <hrn-game nr="17" credits="true"></hrn-game>
</body>
</html>

Automation

I/O Management When active, console output within the sandbox is silenced to keep the host clean.
Lifecycle Sync Uses visibilitychange to pause games when users switch tabs or minimize windows.
Lazy Loading Uses IntersectionObserver to only load games once they enter the viewport.
