hrn.db.min.js

HyperRush's advanced data engine. Specialized logic for sub-millisecond game retrieval and metadata processing.

Import

Include the minified library directly in your project via JSDelivr CDN.

<!-- Global Database Engine -->
<script src="https://cdn.jsdelivr.net/gh/HyperRushNet/dev/lib/hrn.db.min.js"></script>

Quick Start

Once imported, the engine initializes the global hrn.db object. All methods are asynchronous.

// Find Action games optimized for Desktop
const results = await hrn.db.find({
   category: "Action",
   device: "pc-no-touch" 
});

Core Methods

• find(criteria)

Executes an exact match (AND logic) across the database. Supports array-based fields like device.

• search(query)

Broad fuzzy search scanning names and categories. Automatically sorts results by relevance.

• getMeta()

Returns database statistics and unique filter values for your UI components.

{
  "total": 42,
  "categories": ["Action", "Puzzle", "Sports"],
  "devices": ["phone", "pc-no-touch"]
}

Engine Features

Encapsulation Logic is fully self-contained within the hrn namespace to avoid conflicts with other scripts.
Lazy Caching Remote data is fetched only once on the first call and stored in a private memory buffer.
Pure Architecture Built without DOM dependencies, making hrn.db.min.js compatible with any JS environment.
