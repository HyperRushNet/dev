Game Database

HyperRush's advanced data engine. Specialized logic for sub-millisecond game retrieval and metadata processing.

Core Methods

• find(criteria)

Filters the database based on exact matches. Support for case-insensitive array checks (e.g., device).

• search(query)

Fuzzy search across names, categories, and devices. Automatically sorts by relevance (index-based).

• all()

Returns the complete database as an array of objects.

• sort(data, key, reverse)

Utility to sort any data set by a specific key. reverse is a boolean.

• getMeta()

Aggregates unique values for your interface.

Engine Features

Internal Fetcher Automated lazy-loading from the central games.json storage with built-in error handling.
Case Insensitive All queries are automatically normalized, ensuring matches regardless of user input casing.
Memory Caching The t = !0 flag zorgt ervoor dat de JSON payload slechts eenmaal per sessie wordt gedownload.

Implementation Example

Using top-level await within a module to fetch, filter, and analyze the database.



<script type="module">
  // Fetch all entries
  const fullData = await hrn.db.all();

  // Filter by specific category
  const oneVOneFind = await hrn.db.find({ category: "Horror" });

  // Fuzzy search for keywords
  const oneSearch = await hrn.db.search("1");

  // Sort full dataset by name (A-Z)
  const sorted = await hrn.db.sort(fullData, "name", false);

  // Get database statistics
  const meta = await hrn.db.getMeta();

  console.log(fullData, oneVOneFind, oneSearch, sorted, meta);
</script>
